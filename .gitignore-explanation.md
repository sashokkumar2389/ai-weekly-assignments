# .gitignore Configuration Guide

## Overview
This document explains what files and folders are excluded from git and why node_modules should **NEVER** be pushed to GitHub.

---

## 🚫 What Should NOT Be Pushed to GitHub

### 1. **node_modules/** (CRITICAL)
- **Size**: 50MB - 500MB+ per project
- **Reason**: Developers install dependencies locally using `npm install`
- **Best Practice**: Only commit `package.json` and `package-lock.json`
- **GitHub Impact**: Dramatically increases repository size and cloning time

### 2. **Environment Variables (.env files)**
- Contains API keys, passwords, database credentials
- **Never commit**: `.env`, `.env.local`, `.env.*.local`
- Use `.env.example` template instead

### 3. **Build Outputs**
- `dist/` - Production builds
- `build/` - Build artifacts
- `*.js.map` - Source maps
- These are generated during build, not source code

### 4. **IDE & Editor Files**
- `.vscode/` - VS Code settings
- `.idea/` - IntelliJ settings
- `*.swp`, `*.swo` - Vim temporary files

---

## ✅ What SHOULD Be Pushed to GitHub

### Source Files
- `package.json` - Lists all dependencies
- `package-lock.json` - Locks exact versions for reproducibility
- `tsconfig.json` - TypeScript configuration
- `src/` - Source code
- `docs/` - Documentation

### Configuration Templates
- `.env.example` - Template showing required environment variables
- `.vscode/settings.json` - Shared team settings (optional)

## ⚠️ Common Mistakes to Avoid

### ❌ DON'T:
```bash
git add node_modules/
git push origin main
```

### ✅ DO:
```bash
# Ensure .gitignore exists and contains node_modules/
# Then install locally
npm install

# Commit only package files
git add package.json package-lock.json
git commit -m "Update dependencies"
git push origin main
```

## 🔐 Protecting Secrets

### ✅ Correct Workflow:
1. Create `.env.example` with template values
2. Add `.env` to `.gitignore`
3. Never commit actual `.env` file
4. Share `.env.example` with team members

### Example `.env.example`:
```env
# Groq API
GROQ_API_KEY=your_groq_api_key_here
LLM_MODEL=meta-llama/Llama-3.1-70b-Instruct

# MongoDB
MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/?appName=Cluster0
MONGODB_DB_NAME=db_resumes

# Mistral API
MISTRAL_API_KEY=your_mistral_api_key_here
```
